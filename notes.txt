Uning uchun yangi ijro contexti yaraladi va Call stack ga tushadi. Har bir Exucation context Call stackga tushadi.


Avvalo masalan bir kod misolida ko'rsak

const firstName = "Lydia";
const lastName = "Haliie";

function greet(nameToGreet) {
    const fullName = nameToGreet + " " + lastName;
    return "Hello "+fullName;
}

greet(firstName);

Exucation context 2ga bo'linadi
GEC - Global Exucation context => BU birinchi marta JS falini yuklaganda yaratiladigan standart context. Hech qsysi funksiya ichida bo'lmagan kod shu yerda ishlaydi

FEC - Function Exucation context => Har safar funksiya chaqirilganda yangi FEC(context) yratiladi. Har bir funksiya chqirilishi alohida contextga ega. Hatto bir xil funksiya bir necha marta chaqirilsa ham. Va funksiya ishlagach bu o'chirib yuboriladi.

Va bu har bir GEC va FEC ham 2 ga bo'linadi

Creation Phase - JS Engine ishga tushishidan oldin bajariladigan tayorgarlik bosqichi.
1. O'zgaruvchilar uchun xotrad ajratiladi.
2. Funksiya xotiraga saqlanadi.
3. scope chainlarni hamaniqlashtirib oladi.
4. shu orqali this ni ham aniqlaydi.
this bu meni kim chqirdiga javob beradi . Va shu chaqirgan object malumotlariga this orqali murojat qilamiz.

Exucation context - Engine kodlarni satrma satr bajaradi. var bilan e’lon qilingan o‘zgaruvchilarga
haqiqiy qiymatlar beriladi. let va const o‘zgaruvchilar ishlatila boshlaydi.
Funksiya chaqirilsa — yangi Function Execution Context (FEC) yaratiladi. Kod yuqoridan pastga qarab bajariladi

Call Stack — bu LIFO (Last In, First Out) prinsipida ishlaydigan stack.
1. Avval Global Exucation Context(GEC) stackga qo'shiladi.
2. Funksiya chaqiriladi. Va FEC yaratilib. Stack tepasiga qo'yiladi.
3. Eng tepada turgan context bajariladi.
4. Funksiya tugasa stackdan chiqariladi.
5. Oxirida GEC ham chiqadi.
